# React Learning Course (Based on Chauffeur.Web)

This is a **project-based React course for beginners** built directly on the `src/Chauffeur.Web` app in this repository.

You will learn React by reading and improving a real codebase (routes, forms, dashboard, API layer, styling, tests), not toy examples.

---

## Course philosophy

- **Learn by mapping concepts to files**.
- **Learn in modules** with objectives and practical tasks.
- **Learn with answer keys** (every module now includes solutions).
- **Grow progressively** from fundamentals to architecture-level thinking.

---

## Who this is for

- Beginners who know basic JavaScript and HTML/CSS.
- Junior developers who can build simple UIs but want stronger React architecture understanding.
- Teammates onboarding onto the Chauffeur.Web project.

---

## Prerequisites

- JavaScript fundamentals (variables, functions, arrays/objects)
- Basic HTML/CSS
- Optional: TypeScript basics (types/interfaces)

---

## What you will build confidence in

By the end of this course, you should be able to:

1. Structure a React app with reusable components and shared layout.
2. Build route-based flows with nested pages.
3. Manage local state, derived state, and shared app state with context.
4. Build robust forms and input validation UX.
5. Integrate UI with API abstraction layers (and mock APIs).
6. Apply scalable styling and accessibility improvements.
7. Write baseline tests and plan feature-quality checks.

## Suggested study plan

- **4-day intensive**: 2 modules/day.
- **8-day standard**: 1 module/day.
- **2-week deep mode**: 1 module every 1–2 days + implement challenge tasks.

Tip: for each module, spend 60% time reading and 40% applying changes in code.

---

## How each module is structured

- Goal and outcomes
- Core concepts
- “In this project” file map
- Exercises
- **Solutions / answer key**
- Optional challenge

---

## Completion rubric

You are “course complete” when you can:

- Explain the data flow from page -> context -> API client -> mock API.
- Add a new page and wire navigation.
- Add a feature with state + form + async API call.
- Write one function test and one UI behavior test.
